# Reference: fayazara/macos-app-skills ‚Äî build patterns to adopt (and gaps)

Source: https://github.com/fayazara/macos-app-skills ‚Äî a set of AI-agent "skills" (markdown playbooks + a few reference Swift files + one Go CLI) for shipping native macOS SwiftUI+AppKit apps. **License caveat:** README says "MIT" but there is **no LICENSE file** in the repo (GitHub license API returns null) ‚Üí **reimplement the patterns, don't copy the Swift verbatim**; the Go release CLI is cleaner to vendor. Unmaintained one-day snapshot (created+pushed 2026-05-27); verify any macOS 26 API against the shipping SDK.

## Adopt directly (de-risks our build)
- **Overlay `NSPanel` recipe (our Metal warmth layer):** `styleMask:[.borderless,.nonactivatingPanel]`, `isOpaque=false`, `backgroundColor=.clear`, `hasShadow=false`, `hidesOnDeactivate=false`, `isFloatingPanel=true`, `ignoresMouseEvents=true` (clicks pass through ‚Äî essential), `collectionBehavior=[.canJoinAllSpaces,.fullScreenAuxiliary]`, `level = NSWindow.Level(rawValue: Int(CGShieldingWindowLevel()))`. One panel per `NSScreen` (use `screen.frame` to cover menu bar too). `sharingType=.none` is the screenshot-exempt knob (decide per the reveal-true-color/screenshot toggle). `NSScreen ‚Üí CGDirectDisplayID` bridge (the skill omits it): `screen.deviceDescription[NSDeviceDescriptionKey("NSScreenNumber")]`.
- **Sparkle timing:** create `SPUStandardUpdaterController(startingUpdater:false)` as a stored property on AppDelegate (early init), call `start()` in `applicationDidFinishLaunching`. Use `ObservableObject`+`@Published` with a Combine KVO bridge (`updater.publisher(for:\.canCheckForUpdates)`), **not** `@Observable`. `#if DEBUG` guards on `start()`/`checkForUpdates()`. **Menu-bar-only fix:** before showing the update window, `NSApp.setActivationPolicy(.regular); NSApp.activate(ignoringOtherApps:true)`. Info.plist: `SUFeedURL` (raw GitHub URL), `SUPublicEDKey`, `SUEnableInstallerLauncherService=true`. Hardened Runtime needs no special Sparkle entitlements (2.x).
- **Liquid Glass Settings window:** you can't get glass chrome from a SwiftUI `Window` scene (`.fullSizeContentView` must be set at window creation; SwiftUI resets it). Use a programmatic `NSWindowController` singleton hosting SwiftUI via `NSHostingController`, `styleMask:[.titled,.closable,.resizable,.miniaturizable,.fullSizeContentView]`. Three Form modifiers for glass to show through: `.formStyle(.grouped)`, `.scrollContentBackground(.hidden)`, `.contentMargins(.top,8,for:.scrollContent)`. macOS 26 guard pattern: `if #available(macOS 26.0,*){ .scrollEdgeEffectStyle(.soft,for:.all) }`.
- **Reference-counted activation policy:** an `AppActivationPolicy.enter()/leave()` helper (counter) shared by the Settings window + Sparkle update window so a menu-bar-only (`.accessory`) app foregrounds correctly and flips back on close.
- **Menu-bar icon:** `statusItem.button?.image.isTemplate = true` is CRITICAL (else invisible in light mode).
- **Release CLI:** fork the repo's **Go `release` CLI** (`release/cli`) ‚Äî reads a `release.json`, finds Sparkle's `sign_update`, reads version/build/keys from the exported app via `plutil`, warns on duplicate build numbers, creates+signs the DMG, updates the appcast (preserving entries), commits/pushes, and `gh release create`s. Most reusable asset; Go = license-cleaner to vendor.
- **`create-dmg` flags** (branded DMG): `--volname --background <png> --window-pos --window-size --icon-size --icon "App.app" X Y --app-drop-link X Y --hide-extension "App.app"`.
- **Global hotkey:** Carbon `RegisterEventHotKey` is the only true-global path; `sindresorhus/KeyboardShortcuts` wraps it (no Accessibility permission needed). Carbon callback runs on an arbitrary thread ‚Üí `DispatchQueue.main.async`. Note: global `NSEvent` monitors are read-only (can't consume keys) ‚Äî fine for our hold-to-reveal since we don't suppress.

## Gaps it does NOT cover ‚Üí first-class work for us
- CLI **notarization** (`notarytool submit --wait`, `stapler staple`, `codesign --options runtime`) ‚Äî repo uses the Xcode GUI only.
- **GitHub Actions CI** ‚Äî none; the entire signing/notary/secrets pipeline is on us.
- **Homebrew cask** ‚Äî never mentioned.
- **DDC / IOKit / private-API entitlements + notarization implications** ‚Äî zero coverage (our hardest engine parts).
- Actual Liquid Glass **material APIs** (`glassEffect`/`GlassEffectContainer`) ‚Äî only window chrome is covered; go to Apple macOS 26 docs.
- **Hold-to-reveal (key-up)** semantics ‚Äî only key-down registration shown.
