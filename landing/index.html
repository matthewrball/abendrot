Audit the engine

Don't take our word for it. Read the warmth engine.

After the NightOwl incident, "just trust the menu-bar app" stopped being enough. So the whole engine is public. Here's the actual shape of the API the app drives — the same WarmthEngine that decides, per display, whether to warm via real hardware (DDC), the system gamma path, or a universal overlay — and reports which.

  • Overlay is always the safe default — the engine never silently no-ops.
  • Every display reports its method: Hardware / Gamma / Overlay.
  • No Accessibility, no Screen Recording, no account. Core function needs no permission.
WarmthEngine.swift WarmthKit · v0 contract 
// The public actor the app drives. Overlay is the safe default;
// DDC is opt-in per display; the engine reports the method per screen.
public actor WarmthEngine {
    public func setWarmth(_ level: WarmthLevel) async
    public func setScheduleMode(_ mode: ScheduleMode) async

    // Reveal True Color — suspend warmth across ALL displays, then ease back.
    public func beginReveal() async
    public func endReveal() async

    // Per-display: opt in to real hardware warmth, or force a layer.
    public func setHardwareDDCEnabled(_ on: Bool, for id: DisplayIdentity) async

    // Emergency restore — always available. Displays are always recoverable.
    public func restoreAllDisplays() async
}

// Which layer is warming a display right now — drives the on-screen badge.
public enum DisplayMethod: String {
    case hardware   // DDC RGB-gain — real hardware warmth
    case gamma      // classified, not measured by default
    case overlay    // universal Metal veil — the safe default
}
Control it from your terminal — or your AI

Read the engine. Then drive it from the command line.

The same engine you can audit ships a real CLI. abendrot talks to the running app — set the warmth, switch the schedule, exclude an app, or pop true color — straight from a shell script, a keybinding, or an AI assistant like Claude Code, Codex, or Cursor. And because state comes back as JSON, you can pipe it anywhere.

  • Scriptable: abendrot set warmth 0.8, abendrot reveal --hold 10, abendrot on.
  • Machine-readable: abendrot status --json reports live state, per-display method and all.
  • Local by design: same macOS user, your session, visual state only — no network listener, no privileged helper.
~/zsh abendrot CLI · v1 
# Warm the screen to 80% — or hand the same command to your AI assistant.
$ abendrot set warmth 0.8

# Read the live state back as JSON — pipe it anywhere.
$ abendrot status --json
{
  "enabled": true,
  "mode": "sunset",
  "warmth": 0.8,
  "kelvin": 2700,
  "warmingNow": true,
  "displays": [
    { "name": "Built-in", "method": "overlay" },
    { "name": "LG UltraFine", "method": "hardware" }
  ]
}

# Momentary true-color peek for color-critical work, then it eases back.
$ abendrot reveal --hold 10

CLI shown is the v1 command surface (in development). The JSON above illustrates the shape of abendrot status --json; exact fields are finalized with the shipping build. An official Abendrot MCP server is on the roadmap ("MCP coming") — until then, the AI integration is this CLI, which an assistant runs exactly as you would.

The science, hedged

A small, sensible nudge — not a magic sleep button.

Warmer, dimmer light in the evening means less of the short-wavelength light that tells your brain it's daytime. We link the research rather than asserting outcomes. Abendrot is a general-wellness tool, not a medical device, and makes no medical claims.

Your eyes have a third light sensor

Beyond rods and cones, a non-visual sensor (melanopsin, in ipRGCs) is most sensitive to blue light around ~480 nm — it helps tell your brain whether it's day or night.

Lucas et al., 2014

It's the blue content, not just brightness

What drives the circadian effect of a display is its melanopic (short-wavelength) content more than overall brightness — which is also why warming without dimming blunts the benefit.

Schoellhorn et al., 2023

There's no one "right" setting

Sensitivity to evening light varies more than 50-fold between healthy adults — so we offer a default to personalize, and never imply a single number is "safe for everyone."

Phillips et al., 2019

Screen blue light won't damage your eyes

Ophthalmologists find no good evidence that screen blue light damages your eyes. Tired eyes come mainly from reduced blinking — blink, take breaks, try the 20-20-20 habit.

Am. Academy of Ophthalmology, 2024

The plan, in the open

The road to v1.0.

Abendrot is pre-release and built in public. Here's the honest state of each piece — what's in hand, what's being hardened, and what lands at 1.0. Watch the repo to get the build the moment it ships.

  1. The warmth engine

    In hand

    The layered WarmthKit engine — DDC hardware, system gamma, and the universal Metal overlay, best-available-wins per display — is written and headless-tested. It's the public API you can read in Audit the engine.

  2. Hardening across real displays

    In progress

    Verifying warming on physical built-in and external panels, confirming the per-display method badges (Hardware / Gamma / Overlay) report honestly, and proving the DDC restore tooling before it's enabled by default.

  3. Measured performance numbers

    In progress

    The bundle-size, RAM, and idle-CPU figures shown as Target / Preliminary get replaced with real measurements from a release build before anything is published as fact.

  4. CLI + AI control

    In progress

    The abendrot command-line tool — set warmth and schedule, read status --json, exclude apps, trigger a momentary reveal — so the terminal and AI assistants can drive it. See Control it from your terminal. An official Abendrot MCP server is a planned fast-follow ("MCP coming").

  5. Signed & notarized 1.0 release

    At 1.0

    A signed, notarized .dmg for macOS 26 "Tahoe" on Apple Silicon, with the first-run experience and the hold-to-Reveal True Color hotkey in place. No account, no paywall, free forever.

This roadmap describes work in progress, not shipped features. There is no download yet — the most useful thing you can do today is star and watch the repo so the 1.0 build reaches you the day it lands.
