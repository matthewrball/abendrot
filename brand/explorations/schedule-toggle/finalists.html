Motion audit · A3

Lenses: Emil Kowalski (primary) · Jakub Krehel (secondary) · Jhey Tompkins (selective). macOS utility = restraint first.

Emil — restraint & speed

The functional change (fill + ink) is ~140–220ms warm ease — fast and certain. The decorative flourish runs ~400–460ms but is separable garnish, not the state change, so it's fine. One flag: the flourish re-fires on each keyboard arrow; per "don't animate keyboard-initiated actions," gate it to fire on commit (Enter/Space) only.

Jakub — production polish

Lit-glass selected state (sunset fill + specular sheen + hairline rim + ember glow), press scale, hover lift — reads as finished macOS Tahoe glass. Rays bloom from scale .6 (not 0), so no pop. Clean.

Jhey — craft

The clip-masked sun dipping below the horizon and the staggered ray bloom are genuinely delightful and on-theme without tipping into gimmick. The infinite spin is gone — good call.

SeverityItem
PASSprefers-reduced-motion (global + belt-and-braces), real radiogroup + full keyboard, transform/opacity-only, custom warm-ease curves, --indigo ink contrast.
MINORGlyph flourish re-fires on rapid keyboard arrowing — fire only on commit, or skip glyph anim for keyboard nav.
NOTEFlourish >300ms — acceptable as garnish since the functional selection is fast; keep it that way in the SwiftUI port.

Motion audit · D2

Lenses: Emil Kowalski (primary) · Jakub Krehel (secondary) · Jhey Tompkins (selective).

Emil — restraint & speed

The cap rotation + snap run ~420–520ms with an overshoot spring. Emil would trim toward ~380ms — but this is a deliberately tactile instrument set rarely, and the functional readout + shortcuts update instantly, so the long snap is acceptable garnish. Keyboard sets state immediately; the spin is suppressed under reduced-motion.

Jakub — production polish

Machined-glass knob (specular, knurled grip, hairline rim, depth shadow), lit detents, glow bloom on landing — premium and finished. One real gap: the lead's ember arc is a discrete conic-gradient swap — it cross-fades rather than sweeping. In SwiftUI, port it as an animatable trimmed arc so the fill genuinely travels.

Jhey — craft

The dial is the most distinctive thing in the whole set, and D2.3's living-sun face (dips/blooms inside the knob) is a lovely fusion with A3. Inventive without losing the Apple feel.

SeverityItem
PASSprefers-reduced-motion snaps instantly (no vestibular spin), role=switch + aria-checked + shortcuts, drag via Pointer Events with snap-to-nearer-detent, full keyboard incl. Home/End.
MINORLead arc fill is a conic swap (cross-fades, doesn't sweep) — port as a trimmed/animatable arc in SwiftUI.
MINORD2.2 transitions mask (costly to animate) — fine on Mac GPUs, but prefer D2.1/D2.3 for the build.
NOTE420–520ms snap > Emil's 300ms — intentional tactile feel; keep functional state instant.

My read

The deciding question is how prominent the Schedule choice should feel. It's a setting users pick once and rarely revisit — for that job, A3.1 wins: instantly legible as a two-way choice, most native, lowest port risk, and the living glyph still gives it a small, on-brand moment of delight. It's the safe, correct default.

If you want Schedule to be a signature, look-at-this moment — and the founder's brief did lean "custom / artful" — D2.3 (living-sun dial) is the showpiece, and it folds A3's living glyph into the knob face, so you don't fully lose that idea. The segmented shortcuts keep it honest for first-time users and keyboard.

Either ports cleanly to SwiftUI (gradients, shapes, springs — no raster). Tell me the winner (down to the variation) and I'll build it into ModeControl.swift with the audit fixes applied — or I can prototype both in Swift if you want to feel them on real hardware before committing.
