Menu-bar "vibrant template" · 18px · §21.4

The single most-seen surface. A monochrome NSImage template (adapts to light/dark bars) that subtly glows amber when warming is active, and carries a faint outer glow / 0.5pt safety stroke so it never disappears on warm, busy wallpapers (desktop-tinting).

01

On both system bars — active vs inactive

Inactive = pure template (inherits bar color like every native icon). Active = amber fill + soft outer glow so a glance tells you warming is ON without opening the popover.

Active treatment: the amber fill is for the ACTIVE state only. On a light bar, the glow is tightened and the fill leans to --accent-deep so it stays legible against bright menu bars — never washed out.
02

Survives desktop-tinting — warm & busy wallpapers

macOS tints the translucent menu bar with the wallpaper behind it. A warm icon on a warm wallpaper is the failure case. The faint outer glow (active) and the 0.5pt dark safety stroke (always) keep the mark separated from the background.

Why this works: a pure-amber fill alone would vanish on these. The 0.5pt semi-opaque dark stroke on the disc + a 2px white/dark dual outer glow create a constant local contrast halo, so the mark reads on hot-orange, golden, photographic, and high-frequency backgrounds alike.
03

The template master

The 18×18 template glyph (the A1 half-sun + horizon — the convergence target from icons-round-1.html, degraded to its menu-bar form). Ship as a PDF/SVG template image; macOS recolors it per appearance. Glow + active fill are applied in code, not baked into the template.

template @ rest
<!-- abendrot-menubar-template.svg  (18×18 @1x; export @2x/@3x) -->
<svg width="18" height="18" viewBox="0 0 24 24"
     xmlns="http://www.w3.org/2000/svg" fill="none">
  <!-- half-sun (filled) -->
  <path d="M7 15 a5 5 0 0 1 10 0 Z" fill="#000"/>
  <!-- horizon -->
  <line x1="4" y1="15" x2="20" y2="15"
        stroke="#000" stroke-width="2" stroke-linecap="round"/>
</svg>

// AppKit — INACTIVE: mark it a template; macOS handles light/dark monochrome:
let img = NSImage(named: "abendrot-menubar-template")!
img.isTemplate = true            // inactive: system recolors it per appearance
statusItem.button?.image = img
statusItem.button?.contentTintColor = nil   // let the template mask own the color

// The two states are EXPLICIT and mutually exclusive:
//   INACTIVE -> img.isTemplate = TRUE  (system monochrome mask, no ember, no glow)
//   ACTIVE   -> img.isTemplate = FALSE (REQUIRED: a template image is force-
//               monochromed by AppKit, so contentTintColor/glow would be ignored
//               while isTemplate stays true). With isTemplate = false you then set:
//                 button.contentTintColor = NSColor(ember)   // amber fill
//                 + a CALayer shadow/glow (radius ~3, ember @ 0.85) on the button
//                 + the baked 0.5pt dark safety stroke variant (wallpaper-tinting).
//
// e.g. on activation:
//   img.isTemplate = false
//   statusItem.button?.image = img
//   statusItem.button?.contentTintColor = NSColor(ember)
//   // add/animate the glow layer here
// on deactivation, flip back: img.isTemplate = true; contentTintColor = nil; drop glow.
Note: keep the glyph on a 24-unit grid but ship the rendered template at native 18px (+@2x/@3x). Stroke width 2 on a 24 grid ≈ 1.5px at 18px — heavy enough to read after wallpaper tinting. Active glow is a layer shadow so Reduce-Transparency/Reduce-Motion users can have it suppressed to a simple fill swap.
